## [1.2.0] - 2024-XX-XX

### Added
- **Arabic language support**: Full support for Arabic speech recognition
- **French language support**: Full support for French speech recognition  
- **Broadcast model**: New model optimized for news and media content
- **Word-level confidence scores**: Get confidence scores for individual words

### Improved
- Cantonese recognition accuracy improved by 3%
- Reduced streaming latency by 15%
- Better handling of background noise in telephony model

### Fixed
- Fixed issue with speaker diarization incorrectly merging speakers
- Fixed punctuation placement in code-switched utterances
- Fixed WebSocket reconnection handling

---

## [1.1.0] - 2024-XX-XX

### Added
- **Speaker diarization**: Identify up to 8 different speakers
- **Keyword biasing**: Boost recognition of domain-specific terms
- **Vietnamese language support**: Full support for Vietnamese
- **Thai language support**: Full support for Thai
- **Async transcription**: Submit jobs and receive results via webhook

### Improved
- Mixed-language detection accuracy improved by 5%
- Reduced API response times by 20%
- Better handling of overlapping speech

### Fixed
- Fixed timestamp drift in long streaming sessions
- Fixed issue with special characters in keywords
- Fixed rate limit headers not updating correctly

---

## [1.0.0] - 2024-XX-XX

### Initial Release

The first public release of the Fano Speech API.

#### Features
- **Pre-recorded transcription**: Transcribe audio files up to 2 hours
- **Real-time streaming**: WebSocket-based streaming transcription
- **Multilingual support**: English, Cantonese, Mandarin, Hokkien, Bahasa Malaysia, Bahasa Indonesia
- **Mixed-language detection**: Automatic code-switching detection
- **Automatic punctuation**: Smart punctuation placement
- **Multiple models**: General, telephony, and meeting models

#### Supported Audio Formats
- WAV
- MP3
- FLAC
- OGG
- WebM

## Migration Guides

<AccordionGroup>
  <Accordion title="Migrating from Beta to v1.0.0">
    ### Breaking Changes
    
    1. **Authentication**: API key format changed
       - Old: `X-API-Key: key`
       - New: `Authorization: Bearer key`
    
    2. **Response format**: Standardized response structure
       - All responses now wrapped in `{"success": true/false, "data": {...}}`
    
    3. **Endpoint changes**:
       - `/transcribe` → `/v1/transcribe`
       - `/stream` → `/v1/stream`
    
    ### Migration Steps
    
    1. Update authentication headers
    2. Update endpoint URLs to include `/v1/`
    3. Update response parsing for new structure
  </Accordion>
</AccordionGroup>
