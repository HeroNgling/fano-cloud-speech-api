<Note>
Results are pushed automatically by the server as audio is processed. You don't need to request them.
</Note>

## Result Types

<AccordionGroup>
  <Accordion title="Interim Results (isFinal: false)" icon="spinner">
    Tentative transcriptions that may change as more audio context is received.
    
    - Confidence is typically `0.0` for interim results
    - Useful for real-time UI feedback
    - Will be replaced by final results
    
    ```json
    {
      "results": [{
        "alternatives": [{
          "transcript": "Hello, how can I",
          "confidence": 0.0,
          "startTime": "0.000s",
          "endTime": "1.500s",
          "languageCode": "en"
        }],
        "isFinal": false
      }]
    }
    ```
  </Accordion>
  
  <Accordion title="Final Results (isFinal: true)" icon="check">
    Committed transcriptions that won't change.
    
    - Includes confidence score (0.0 - 1.0)
    - Safe to store and display permanently
    - No further updates for this audio segment
    
    ```json
    {
      "results": [{
        "alternatives": [{
          "transcript": "Hello, how can I help you today?",
          "confidence": 0.95,
          "startTime": "0.000s",
          "endTime": "3.200s",
          "languageCode": "en"
        }],
        "isFinal": true
      }]
    }
    ```
  </Accordion>
</AccordionGroup>

## Response Schema

| Field | Type | Description |
|-------|------|-------------|
| `results[].alternatives[].transcript` | string | Transcribed text |
| `results[].alternatives[].confidence` | float | Confidence score (0.0-1.0) |
| `results[].alternatives[].startTime` | string | Start time (e.g., "0.000s") |
| `results[].alternatives[].endTime` | string | End time (e.g., "3.200s") |
| `results[].alternatives[].languageCode` | string | Detected language code |
| `results[].resultEndTime` | string | Result end time |
| `results[].channelTag` | integer | Channel identifier |
| `results[].isFinal` | boolean | `true` = final, `false` = interim |
| `metadata.asrModels` | string[] | ASR models used |

## Processing Results

<Tabs>
  <Tab title="Python">
```python
async def receive_transcripts(ws):
    async for message in ws:
        response = json.loads(message)
        
        # Check for errors
        if "error" in response:
            print(f"Error: {response['error']['message']}")
            continue
        
        # Process results
        if "results" in response:
            for result in response["results"]:
                alt = result["alternatives"][0]
                transcript = alt["transcript"]
                is_final = result.get("isFinal", False)
                
                if is_final:
                    print(f"[FINAL] {transcript}")
                    # Store this transcript permanently
                else:
                    print(f"[INTERIM] {transcript}", end="\r")
                    # Update UI but don't store
```

## Understanding Confidence Scores

| Score Range | Interpretation |
|-------------|----------------|
| `0.0` | Interim result (confidence not calculated) |
| `0.0 - 0.5` | Low confidence - may need review |
| `0.5 - 0.8` | Medium confidence - generally reliable |
| `0.8 - 1.0` | High confidence - very reliable |

<Warning>
Confidence scores are estimates and should not be relied upon for critical decisions. A value of `0.0` indicates confidence was not set (typically for interim results).
</Warning>

## Best Practices

1. **Always check `isFinal`** before storing transcripts
2. **Handle errors gracefully** - check for `error` field in responses
3. **Use interim results for UI feedback** only, not for storage
4. **Process results asynchronously** to avoid blocking audio sending
