## API Key Authentication

The Fano Speech API uses API key authentication. You must include your license key in the header of every request.

### Header Format

| Header | Value |
|--------|-------|
| `Fano-License-Key` | Your API key |

### Example Request

<CodeGroup>

```bash cURL
curl -X POST https://studio.fano.ai/api/v1/transcript \
  -H "Fano-License-Key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"config": {...}, "audio": {...}}'
```

```python Python
import requests

headers = {
    "Fano-License-Key": "YOUR_API_KEY",
    "Content-Type": "application/json"
}

response = requests.post(
    "https://studio.fano.ai/api/v1/transcript",
    headers=headers,
    json=payload
)
```

```javascript JavaScript
const response = await fetch("https://studio.fano.ai/api/v1/transcript", {
  method: "POST",
  headers: {
    "Fano-License-Key": "YOUR_API_KEY",
    "Content-Type": "application/json"
  },
  body: JSON.stringify(payload)
});
```

</CodeGroup>

## Getting Your API Key

To obtain your API key:

1. Log in to the [Fano Speech API Portal](https://app.fanocloud.com)
2. Navigate to **API Keys**
3. Copy your credentials

<Warning>
**Security Best Practices:**
- Never expose your API key in client-side code
- Store API keys in environment variables
- Rotate your keys periodically
- Use different keys for development and production
</Warning>

## Error Responses

If authentication fails, you will receive a `401 Unauthorized` response:

```json
{
  "error": {
    "code": 401,
    "message": "Unauthorized"
  }
}
```

Common causes:
- Missing `Fano-License-Key` header
- Invalid or expired API key
