# Authentication

The Fano Speech API uses API keys to authenticate requests. This guide explains how to obtain and use your API keys securely.

## Getting Your API Key

<Steps>
  <Step title="Sign Up">
    Create an account at [dashboard.fano.ai](https://dashboard.fano.ai)
  </Step>
  
  <Step title="Navigate to API Keys">
    Go to **Settings** → **API Keys** in your dashboard
  </Step>
  
  <Step title="Create New Key">
    Click **Create API Key** and give it a descriptive name
  </Step>
  
  <Step title="Copy Your Key">
    Copy the key immediately - it won't be shown again
  </Step>
</Steps>

## Using Your API Key

### HTTP Header Authentication

Include your API key in the `Authorization` header:

<CodeGroup>

```python Python
import requests

headers = {
    "Authorization": "Bearer your_api_key_here"
}

response = requests.post(
    "https://api.fano.ai/v1/transcribe",
    headers=headers,
    json={"audio_url": "https://example.com/audio.wav"}
)
```

```javascript JavaScript
const response = await fetch('https://api.fano.ai/v1/transcribe', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your_api_key_here',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    audio_url: 'https://example.com/audio.wav'
  })
});
```

```bash cURL
curl -X POST "https://api.fano.ai/v1/transcribe" \
  -H "Authorization: Bearer your_api_key_here" \
  -H "Content-Type: application/json" \
  -d '{"audio_url": "https://example.com/audio.wav"}'
```

</CodeGroup>

### WebSocket Authentication

For streaming connections, pass the API key as a query parameter:

```javascript
const ws = new WebSocket('wss://api.fano.ai/v1/stream?api_key=your_api_key_here');
```

Or use a header-based approach with libraries that support it:

```python
import websocket

ws = websocket.create_connection(
    "wss://api.fano.ai/v1/stream",
    header=["Authorization: Bearer your_api_key_here"]
)
```

## API Key Security

<Warning>
**Never expose your API keys in client-side code, public repositories, or logs!**
</Warning>

### Best Practices

<AccordionGroup>
  <Accordion title="Use Environment Variables" icon="terminal">
    Store API keys in environment variables instead of hardcoding:
    
    ```python
    import os
    
    API_KEY = os.environ.get('FANO_API_KEY')
    ```
    
    ```bash
    export FANO_API_KEY="your_api_key_here"
    ```
  </Accordion>
  
  <Accordion title="Use Secrets Management" icon="lock">
    For production, use a secrets manager:
    - AWS Secrets Manager
    - Google Cloud Secret Manager
    - Azure Key Vault
    - HashiCorp Vault
  </Accordion>
  
  <Accordion title="Rotate Keys Regularly" icon="arrows-rotate">
    Create new API keys periodically and deprecate old ones:
    1. Create a new key
    2. Update your applications
    3. Delete the old key
  </Accordion>
  
  <Accordion title="Use Key Restrictions" icon="shield">
    Where possible, restrict API keys to:
    - Specific IP addresses
    - Specific endpoints
    - Rate limits
  </Accordion>
</AccordionGroup>

## Managing API Keys

### Viewing Keys

View all your API keys in the dashboard. For security, only the last 4 characters are shown.

### Revoking Keys

To revoke a compromised key:

1. Go to **Settings** → **API Keys**
2. Find the key to revoke
3. Click **Revoke**
4. Confirm the action

<Tip>
Create a new key before revoking the old one to avoid service interruption.
</Tip>

## Authentication Errors

| Error | Cause | Solution |
|-------|-------|----------|
| `UNAUTHORIZED` | Missing or invalid API key | Verify your API key is correct |
| `KEY_REVOKED` | API key has been revoked | Create a new API key |
| `KEY_EXPIRED` | API key has expired | Renew or create a new key |
| `IP_NOT_ALLOWED` | Request from non-whitelisted IP | Update IP restrictions |

### Error Response Example

```json
{
  "success": false,
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Invalid API key provided",
    "details": {
      "hint": "Check that your API key is correct and not revoked"
    }
  }
}
```
