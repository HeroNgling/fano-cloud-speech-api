# Keyword Biasing

Keyword biasing allows you to improve the recognition accuracy of specific words or phrases that are important to your use case. This is particularly useful for:

- Industry-specific terminology
- Product names and brand names
- Technical jargon
- Names of people and places

## How It Works

When you provide keywords, the speech recognition model increases the probability of recognizing these words when they appear in the audio. This helps overcome the natural bias toward more common words that sound similar.

<Info>
For example, "Fano" might be misrecognized as "pharaoh" without keyword biasing, but adding "Fano" as a keyword ensures accurate recognition.
</Info>

## Usage

### Pre-recorded Audio

Include keywords in your transcription request:

<CodeGroup>

```python Python
import requests

response = requests.post(
    "https://api.fano.ai/v1/transcribe",
    headers={
        "Authorization": "Bearer your_api_key",
        "Content-Type": "application/json"
    },
    json={
        "audio_url": "https://example.com/audio.wav",
        "keywords": ["Fano", "speech recognition", "diarization"],
        "keyword_boost": 2.0  # Optional: strength of the boost
    }
)
```

```javascript JavaScript
const response = await fetch('https://api.fano.ai/v1/transcribe', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer your_api_key',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    audio_url: 'https://example.com/audio.wav',
    keywords: ['Fano', 'speech recognition', 'diarization'],
    keyword_boost: 2.0
  })
});
```

```bash cURL
curl -X POST "https://api.fano.ai/v1/transcribe" \
  -H "Authorization: Bearer your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "audio_url": "https://example.com/audio.wav",
    "keywords": ["Fano", "speech recognition", "diarization"],
    "keyword_boost": 2.0
  }'
```

</CodeGroup>

### Streaming Audio

Send keywords in the configuration message:

```json
{
  "type": "config",
  "language": "auto",
  "keywords": ["Fano", "speech recognition", "diarization"],
  "keyword_boost": 2.0
}
```

## Parameters

<ParamField body="keywords" type="array" required>
  Array of words or phrases to boost. Maximum 500 keywords per request.
  
  ```json
  ["keyword1", "keyword2", "multi-word phrase"]
  ```
</ParamField>

<ParamField body="keyword_boost" type="number" default="1.5">
  The strength of the keyword boost. Higher values increase the likelihood of recognizing keywords.
  
  - **1.0**: No boost (baseline)
  - **1.5**: Mild boost (default)
  - **2.0**: Moderate boost
  - **3.0**: Strong boost
  
  <Warning>
  Values above 3.0 may cause over-recognition of keywords even when they're not spoken.
  </Warning>
</ParamField>

## Best Practices

<AccordionGroup>
  <Accordion title="Use Specific Terms" icon="bullseye">
    Focus on words that are unique to your domain and might not be recognized correctly by default.
    
    **Good**: `["Fano Labs", "ASR", "diarization"]`
    
    **Not needed**: `["hello", "meeting", "computer"]` (common words already recognized well)
  </Accordion>
  
  <Accordion title="Include Variations" icon="arrows-rotate">
    Add common variations and abbreviations of important terms.
    
    ```json
    ["API", "A.P.I.", "artificial intelligence", "AI"]
    ```
  </Accordion>
  
  <Accordion title="Use Multi-word Phrases" icon="text">
    For compound terms, include both the full phrase and individual words.
    
    ```json
    ["machine learning", "machine", "learning", "ML"]
    ```
  </Accordion>
  
  <Accordion title="Start with Moderate Boost" icon="sliders">
    Begin with the default boost (1.5) and increase only if keywords are still being missed.
  </Accordion>
</AccordionGroup>

## Limits

| Limit | Value |
|-------|-------|
| Maximum keywords per request | 500 |
| Maximum characters per keyword | 100 |
| Maximum total keyword characters | 10,000 |

```json
  "keywords": [

    "myocardial infarction",
    "electrocardiogram",
    "ECG",
    "EKG"
  ],
  "keyword_boost": 2.0
}
```

```json
{
  "keywords": [
    "plaintiff",
    "defendant",
    "habeas corpus",

    "affidavit"
  ],
  "keyword_boost": 1.5
}
```
